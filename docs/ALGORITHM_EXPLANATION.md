# 네비게이션 알고리즘 핵심 개념

## 질문: 도로마다 위험 스코어가 부여되고 가장 위험도가 낮은 경로를 추천하는 게 맞나요?

**답**: **거의 맞습니다!** 하지만 **균형을 잡아** 찾습니다.

---

## 🔍 세부 설명

### 1. 각 도로에 위험 스코어 부여 ✅

맞습니다!

```
도로 그래프 구조:
노드(교차로) ──────────[엣지: 도로구간]────────── 노드
                     ↓
                  위험도 0-100점
```

**예시**:
- 도로 A: 위험도 20점 (검문소 1개)
- 도로 B: 위험도 80점 (무장충돌 지역)
- 도로 C: 위험도 10점 (안전)

---

### 2. 경로 추천 방식: "절대 위험도 최소화" vs "균형 경로"

#### ❌ 순수 위험도만 최소화 (문제점)

만약 **위험도만** 따진다면:

```
출발지 ──[위험도 5]── A ──[위험도 5]── B ──[위험도 5]── 목적지
총: 15점, 거리: 50km, 시간: 90분

출발지 ──[위험도 1]── C ──[위험도 1]── D ──[위험도 1]── E ──[위험도 1]── 목적지
총: 4점, 거리: 100km, 시간: 180분
```

→ 위험도는 더 낮지만 **2배 멀리 돌아감** (현실적이지 않음)

---

#### ✅ 실제 구현: "거리 + 위험도" 균형

실제로는 **가중치 함수**를 사용합니다:

```python
가중치 = 거리(m) + 위험도 × 100

예시:
도로 A: 5000m + 20 × 100 = 7000
도로 B: 10000m + 5 × 100  = 10500
```

→ 도로 A가 더 "저렴"함 (위험도가 낮아서 거리 단점 상쇄)

---

## 📊 실제 예시

### 시나리오: 주바 공항 → 시청

#### 빠른 경로 (거리만 고려)
```
공항 ──[2km, 위험도 90]── 검문소구간 ──[1km, 위험도 70]── 시청
총: 3km, 위험도 평균 80점
소요시간: 8분
```

#### 안전 경로 (거리 + 위험도 고려)
```
공항 ──[4km, 위험도 10]── 우회도로 ──[2km, 위험도 20]── 시청
총: 6km, 위험도 평균 15점
소요시간: 12분
```

**알고리즘 선택**:
- 안전 경로 가중치: 6000m + 15×100 = 7500
- 빠른 경로 가중치: 3000m + 80×100 = 11000

→ **안전 경로**를 추천! (거리는 더 멀지만 위험도가 훨씬 낮아서)

## 🎯 핵심 포인트

### 위험도만 최소화하는 게 아니라...

```
"적당히 안전하면서 적당히 빠른 경로"
```

를 찾습니다.

### 왜 이렇게 하나요?

1. **안전만 따지면**: 너무 멀리 돌아감 (비현실적)
2. **빠르기만 따지면**: 위험 지역 통과 (본질 상실
3. **균형을 잡으면**: 적절한 안전도 + 적절한 거리

---

## 📈 가중치 조정 가능

만약 더 안전을 중시하고 싶다면:

```python
# 현재 (균형적)
가중치 = 거리 + 위험도 × 100

# 더 안전 중시
가중치 = 거리 + 위험도 × 200  # 위험도가 2배 더 중요

# 빠르기 중시
가중치 = 거리 + 위험도 × 50   # 위험도가 덜 중요
```

- 🎛️ 가중치로 밸런스 조절 가능

이렇게 하면 **"현실적인 안전 경로"**를 찾을 수 있습니다!
