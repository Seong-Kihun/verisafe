## 💡 VeriSafe가 해결하는 방법

### 1. 지리정보 부족 해결 → **크라우드소싱 매핑**

**Before (현재):**
```
김현장: Google Maps 보면서 "이게 뭐지?" 고민
         ↓
      현지 직원에게 전화
         ↓
      개인 노트에 메모
```

**After (VeriSafe):**
```
김현장 (Mapper 권한):
  웹 포털 접속
    ↓
  건물 클릭 → "주바 종합병원" 입력 (30초)
    ↓
  검수자(Admin) 승인 (1분)
    ↓
  전 세계 모든 VeriSafe 사용자에게 즉시 공유 ✅
```

**핵심 기능:**
- **Mapper 역할**: 김현장이 직접 지도에 정보 추가
  - Leaflet.draw로 건물/도로 그리기
  - 속성 입력: 이름, 용도, 운영시간, 전화번호
  - 사진 첨부 가능

- **DetectedFeature 모델**: AI + 사람의 협업
  - 위성 이미지 → YOLO v8 → 건물 자동 탐지
  - Mapper가 용도만 입력하면 됨 (좌표는 자동)

- **검수 시스템**: OpenStreetMap 방식
  ```
  Mapper 생성 → review_status='pending'
              ↓
           Admin 검수 → 'approved'
              ↓
           전체 사용자에게 공개
  ```

**효과:**
- ✅ 주바 사무소가 입력한 정보 → 카불 사무소도 즉시 활용
- ✅ 1명이 입력 → 1000명이 혜택
- ✅ 지속적으로 지도 품질 향상

### 2. 안전 경로 탐색 → **통합 솔루션**

**Before:**
```
70분 수작업 (Google Maps + UN 보고서 + Excel)
```

**After:**
```
VeriSafe 앱:
  출발지 입력 (10초)
    ↓
  도착지 입력 (10초)
    ↓
  "안전한 경로" 선택 (5초)
    ↓
  6개 경로 + 위험도 점수 즉시 표시 (3초) ✅
    ↓
  총 28초 소요 (99.3% 시간 단축!)
```

**핵심 알고리즘:**
```python
# K-Shortest Paths + A* 알고리즘
def calculate_safe_route(start, end):
    # 1. 실제 도로 네트워크 (OSMnx)
    graph = load_juba_road_network()

    # 2. 모든 엣지에 위험도 계산 (5분마다 자동)
    for edge in graph.edges():
        edge['risk_score'] = calculate_hazard_proximity(
            edge_center,
            active_hazards  # ACLED + UN OCHA + 사용자 제보
        )

    # 3. 안전 가중치로 경로 계산
    weight = lambda u, v, d: (
        d['length'] * 1000 +      # 거리
        d['risk_score'] * 100     # 위험도 (높을수록 회피)
    )

    # 4. 다양한 경로 제공
    routes = k_shortest_paths(graph, start, end, k=6, weight=weight)

    # 5. 각 경로의 안전 점수 계산 (0-10)
    for route in routes:
        route['safety_score'] = 10 - (avg_risk / 10)

    return routes  # [safe(9.2), balanced(7.8), fast(6.1), ...]
```

**실제 결과 예시:**
```
경로 1 (가장 안전) - 9.2/10점
  거리: 8.5km | 시간: 25분
  경유: 주바대학교 → UN 캠프 → 병원
  위험: 낮음 (무력충돌 지역 10km 이상 우회)
```
