## 1. 문제인식/문제정의

### 1.1 현장/업무 내 문제점

**현재 KOICA 및 사업수행기관의 경로 설정 방식의 문제점:**

개발협력 사업 현장에서 사업수행인력이 안전한 이동경로를 계획할 때, **통일된 시스템이나 프로세스가 부재**한 상황입니다. 각 사업수행기관과 KOICA 현지 사무소는 다음과 같은 방식으로 경로를 설정하고 있습니다:

1. **담당자 개인의 암묵지 의존**
   - 현지 치안 정보를 수집한 후 담당자가 개인 경험과 직관을 바탕으로 경로를 판단
   - 체계적인 위험도 평가 없이 주관적 판단에 의존

2. **표준화된 의사결정 체계 부재**
   - 기관별, 담당자별로 서로 다른 기준과 방식 적용
   - 위험 정보의 신뢰도나 최신성을 검증할 방법 없음
   - 안전 경로 설정에 대한 가이드라인이나 도구 부재

3. **결과적 문제: 사업 활동 지연 및 인력 안전 위협**
   - 분쟁 및 시위 등 불안정한 치안 상황으로 대상지 접근 제한
   - 사업 활동이 지연되거나 중단되는 경우 빈번 발생
   - 사업수행인력의 안전이 체계적으로 보호되지 못함

**구체적 사례:**
- 업무회의에서 "치안불안으로 해당 지역 활동 지연" 보고가 반복됨
- 각 기관에 GIS나 경로 계획 전문가가 없어 담당자가 자의적으로 판단
- 안전 관련 의사결정에 대한 책임 소재가 불명확

### 1.2 원인 및 발생 맥락

#### 원인 1: 개도국 지리정보 인프라 부족

**분쟁취약국의 지리정보 부실 문제:**

1. **기초 지리 정보의 부재**
   - 상용 지도 앱(구글맵 등)에서 건물은 표시되지만 **건물의 용도나 명칭 정보 없음**
   - 비공식 도로, 끊어진 다리, 임시 대피소 등의 정보 누락
   - 지도상에는 도로가 있어도 실제로는 사용 불가능한 경우 빈번

2. **동적 위험 정보의 부족**
   - 불법 검문소, 임시 시위 현장 등 **실시간으로 변화하는 위험 요소** 파악 어려움
   - ACLED(Armed Conflict Location & Event Data), GDACS(Global Disaster Alert and Coordination System) 등 외부 데이터가 있으나 **실시간성 부족 및 세밀도 낮음**
   - 현지 상황에 정통한 사람의 정보가 체계적으로 공유되지 않음

**예시: 남수단 주바 지역**
- OSM(OpenStreetMap) 데이터 기준 도로 정보는 있으나 실제 통행 가능 여부 불명확
- 병원, 학교, 안전 대피소 등의 위치 정보 부정확
- 무장 충돌, 검문소 위치 등 치안 정보는 외부 API에 의존하나 **몇 시간~며칠 지연**

#### 원인 2: 안전경로 탐색 전문 도구 부재

**기존 솔루션의 한계:**

1. **일반 네비게이션 앱의 문제**
   - 구글맵, 카카오맵 등: **"가장 빠른 경로"만 제공**
   - 치안, 위험도를 고려한 경로 계산 기능 전무
   - 개도국 도로 데이터 부정확 및 업데이트 부족

2. **안전 정보 제공 기관의 한계**
   - UN OCHA, ACLED, ReliefWeb 등: **위험 정보만 제공**, 경로 계산 기능 없음
   - 정보와 경로 계획이 분리되어 있어 담당자가 **수동으로 조합**해야 함
   - 정보 제공 시점과 실제 사용 시점 간 시차 발생

3. **상용 프로그램의 부재**
   - **"안전한 경로"를 자동으로 계산해주는 상용 앱이나 서비스가 존재하지 않음**
   - 개도국 특화 내비게이션 솔루션 부재
   - 결과적으로 담당자 개인이 여러 정보 소스를 종합 판단

#### 원인 3: 파편적 정보 유통 및 지식 공유 체계 부재

**조직 차원의 구조적 문제:**

1. **암묵지의 문제**
   - 담당자가 현지에서 수집한 안전 정보는 **개인의 경험으로만 남음**
   - 인수인계 시 체계적 지식 전달 어려움
   - 새로 부임한 담당자는 처음부터 다시 정보 수집

2. **정보 공유 시스템 부재**
   - 기관 간, 담당자 간 위험 정보 공유 플랫폼 없음
   - 같은 지역에서 활동하는 다른 기관의 경험을 활용하지 못함
   - 중복된 정보 수집 노력 발생

3. **통합적 안전 정보 서비스 부족**
   - 여러 출처(UN, NGO, 현지 언론 등)의 정보를 **한 곳에서 확인할 수 없음**
   - 정보의 신뢰도, 최신성 검증 어려움
   - 위험도를 정량화하여 비교할 수단 부재

   - "기관 간 정보 공유 시스템 없어 중복 노력 발생"

### 1.4 현재의 한계

#### 1. 시스템적 한계

**표준화된 안전 경로 계획 프로세스 부재**
- 각 기관, 각 담당자마다 다른 방식으로 경로 결정
- 의사결정 근거가 문서화되지 않아 사후 검증 불가능
- 위험도 평가 기준이 주관적이고 일관성 없음

**정보의 파편화 및 실시간성 부족**
- 여러 출처(UN, NGO, 언론)의 정보를 **수동으로 종합**해야 함
- 정보 수집에 많은 시간 소요
- 정보의 최신성과 신뢰도를 검증할 방법 부재

#### 2. 기술적 한계

**기존 도구의 부적합성**
- 일반 네비게이션 앱: 빠르기만 고려, 안전성 무시
- 위험 정보 플랫폼: 정보 제공만 하고 경로 계산 기능 없음
- **안전한 경로를 자동 계산하는 통합 솔루션 부재**

**개도국 특수성 미반영**
- 글로벌 지도 서비스의 개도국 데이터 부정확
- 비공식 도로, 임시 시설 정보 누락
- 현지 맥락(문화적, 정치적 상황) 미고려

#### 3. 조직적 한계

**지식 관리 체계 부재**
- 개인 경험과 암묵지가 조직 자산으로 전환되지 못함
- 담당자 교체 시 노하우 소실
- 같은 실수 반복 가능성

**기관 간 협력 부족**
- 동일 지역 활동 기관들 간 정보 공유 미흡
- 중복 노력 및 비효율 발생
- 표준화된 협업 플랫폼 부재

#### 4. 결과적 한계

**사업 효율성 저하**
- 안전 문제로 인한 사업 지연 빈번
- 우회 경로 사용으로 시간과 비용 증가
- 현장 접근성 제약으로 사업 목표 달성 어려움

**인력 안전 위협**
- 체계적 안전 관리 부족으로 위험 노출
- 긴급 상황 대응 체계 미흡
- 사업수행인력의 불안감 증대

**데이터 기반 의사결정 불가**
- 정량적 위험도 평가 부재
- 과거 사례 축적 및 분석 미흡
- 예방적 안전 관리보다는 사후 대응 중심

### 2.3 핵심 기능

#### 기능 1: 안전 경로 계산 (Smart Routing)

**문제 해결:**
- 기존: 빠른 경로만 제공 → VeriSafe: **안전 경로 우선 제공**

**작동 원리:**

1. **도로 네트워크 그래프 구축**
   - OpenStreetMap 데이터를 NetworkX 그래프로 변환
   - 각 도로 구간(엣지)에 기본 거리 정보 저장
   - 서버 시작 시 메모리에 로드 (응답 속도 최적화)

2. **위험도 계산 (5분마다 자동 업데이트)**
   ```python
   # 각 도로 구간의 위험도 계산
   위험도 = Σ (주변 위험요소의 위험점수 × 거리 가중치)

   예시:
   - 도로 A: 500m 거리에 무장충돌(위험도 90) → 위험도 45
   - 도로 B: 2km 거리에 검문소(위험도 60) → 위험도 10
   ```

3. **경로 탐색 (A* 알고리즘)**
   ```python
   # 안전 경로 가중치
   가중치 = 거리(m) × 1000 + 위험도 × 100

   예시:
   - 경로 1: 3km, 위험도 20 → 가중치 5,000
   - 경로 2: 5km, 위험도 80 → 가중치 13,000
   → 경로 1 선택 (더 안전함)
   ```

4. **결과 제공**
   - **안전 경로**: 위험도 최소화 우선
   - **빠른 경로**: 거리 최소화 (비교용)
   - 2개 경로를 동시 제공하여 사용자가 선택 가능

**성능:**
- 경로 계산 시간: **1-2초 이내**
- Redis 캐싱으로 반복 요청 99% 성능 향상
- 위험도는 5분마다 백그라운드 업데이트 (실시간성 확보)

#### 기능 2: 다층적 위험 정보 수집

**문제 해결:**
- 기존: 파편화된 정보 출처 → VeriSafe: **통합 플랫폼**

**위험 정보 소스:**

### 2.5 핵심 알고리즘

#### A* 경로 탐색 알고리즘

**선택 이유:**
- Dijkstra보다 빠름 (휴리스틱 사용)
- 최적 경로 보장
- 가중치 커스터마이징 가능

**VeriSafe 커스터마이징:**

```python
# 기본 A* 알고리즘
가중치 = 거리

# VeriSafe 안전 우선 알고리즘
가중치 = 거리 × 1000 + 위험도 × 100

# 예시:
도로 A: 2km, 위험도 80
  → 가중치 = 2000 + 8000 = 10,000

도로 B: 5km, 위험도 10
  → 가중치 = 5000 + 1000 = 6,000
  → 도로 B 선택 (더 안전함)
```

**밸런스 조정:**
- 위험도 계수를 높이면 → 더 안전하지만 멀리 돌아감
- 위험도 계수를 낮추면 → 빠르지만 위험 증가
- 현재 설정은 **적절한 균형** 유지

#### 위험도 계산 알고리즘

```python
# 각 도로 구간의 위험도 계산
def calculate_edge_risk(edge, hazards):
    total_risk = 0
    edge_midpoint = get_midpoint(edge)

    for hazard in hazards:
        distance = haversine_distance(edge_midpoint, hazard.location)

        # 영향 반경 내에 있는지 확인
        if distance <= hazard.radius:
            # 거리 가중치 (가까울수록 높음)
            weight = 1 / (1 + distance)
            total_risk += hazard.risk_score × weight

    return min(total_risk, 100)  # 0-100 범위
```

**특징:**
- 여러 위험 요소의 **누적 효과** 고려
- 거리 기반 가중치로 **현실적 위험도** 반영
- PostGIS 공간 인덱스로 **고속 쿼리** (99.5% 향상)

## 7. 기술적 우수성

### 7.1 성능 최적화

**Before vs After:**

| 항목 | 최적화 전 | 최적화 후 | 개선율 |
|-----|----------|----------|--------|
| 경로 계산 시간 | 25-60초 | 1-2초 | **95%** |
| 위험도 쿼리 | 5-15초 | 0.1초 | **99%** |
| 지도 데이터 로드 | 3-8초 | 0.3초 | **96%** |
| 메모리 사용량 | 2GB | 500MB | **75%** |

**최적화 기법:**
- GraphManager Singleton (그래프 메모리 캐싱)
- Redis 캐싱 (경로 계산 결과)
- PostGIS 공간 인덱스 (위험도 쿼리)
- N+1 쿼리 제거 (SQLAlchemy 최적화)
- CDN 활용 (지도 타일)

### 7.2 확장성

**수평 확장:**
- 무상태(Stateless) API 서버 → 로드 밸런서로 확장 가능
- Redis 클러스터 모드
- PostgreSQL 읽기 복제본

**수직 확장:**
- 그래프 데이터 메모리 관리 최적화
- 비동기 처리 (FastAPI)
- 백그라운드 태스크 큐

**예상 부하:**
- 동시 접속자: 10,000명
- 초당 API 요청: 1,000건
- 데이터베이스 크기: 100GB
- 응답 시간: 평균 2초 이내 유지

### 7.3 보안

**인증/권한:**
- JWT 토큰 기반 인증
- bcrypt 비밀번호 해싱 (salt 10 rounds)
- 역할 기반 접근 제어 (RBAC)

**API 보안:**
- Rate Limiting (DDoS 방지)
- CORS 화이트리스트
- SQL Injection 방지 (ORM 사용)
- XSS 방지 (입력 검증)

**데이터 보안:**
- 개인 정보 암호화
- 프로덕션 환경 시크릿 검증
- HTTPS 강제 (프로덕션)

## 8. 결론

### 8.1 핵심 가치 제안

**VeriSafe는 분쟁취약국 개발협력 현장의 생명을 구하는 플랫폼입니다.**

1. **문제 해결:**
   - 파편화된 안전 정보 → 통합 플랫폼
   - 주관적 경로 판단 → 데이터 기반 자동 계산
   - 개인 암묵지 → 조직 지식 자산

2. **혁신성:**
   - 업계 최초 "안전 우선" 경로 계산
   - AI + 크라우드소싱 + 외부 API 통합
   - 플랫폼 분리 전략으로 사용자 그룹별 최적화

3. **실현 가능성:**
   - MVP 100% 완성 (실제 작동하는 프로토타입)
   - 검증된 기술 스택 (오픈소스 기반)
   - 명확한 비즈니스 모델 (기관 후원 + 프리미엄)

4. **확장성:**
   - 전 세계 개도국으로 확장 가능
   - UN, NGO 등 다양한 조직 활용 가능
   - 오픈소스로 글로벌 커뮤니티 기여

### 8.2 KOICA 미션과의 정렬

**KOICA의 비전:**
> "더 나은 세계를 향한 동행"

**VeriSafe의 기여:**
- ✅ 사업수행인력의 안전 확보 → 지속가능한 개발협력
- ✅ 현장 접근성 향상 → 사업 목표 달성률 증가
- ✅ 데이터 기반 의사결정 → 효율적 리스크 관리
- ✅ 지식 공유 플랫폼 → 조직 학습 및 협업 강화

**SDGs 기여:**
- SDG 3 (건강과 웰빙)
- SDG 11 (지속가능한 도시)
- SDG 16 (평화와 정의)
- SDG 17 (파트너십)

### 8.3 사회적 임팩트

**정량적 임팩트 (3년 목표):**
- 보호 인원: 10,000명 이상의 개발협력 종사자
- 커버리지: 20개국 이상의 분쟁취약국
- 제보 건수: 100,000건 이상의 위험 정보
- 사고 감소: 위험 노출 50% 감소

**정성적 임팩트:**
- 개발협력 현장의 안전 문화 조성
- 글로벌 안전 정보 공유 생태계 구축
- 인도적 가치 실현 (생명 보호)
- 기술로 평화에 기여 (Technology for Peace)

### 8.4 마무리

VeriSafe는 단순한 네비게이션 앱이 아닙니다.

**분쟁취약국에서 일하는 모든 개발협력 종사자들이 안전하게 귀가할 수 있도록 돕는, 생명을 구하는 플랫폼입니다.**

KOICA 아이디어톤을 통해 이 프로젝트가 더 많은 분들의 관심과 지원을 받아, 실제로 현장에서 사용되는 날을 기대합니다.

**"더 나은 세계를 향한 동행, VeriSafe와 함께합니다."**
